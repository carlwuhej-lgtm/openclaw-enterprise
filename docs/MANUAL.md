## 1. 产品概述

### 1.1 产品简介

OpenClaw Enterprise 是专为企业设计的 OpenClaw 集中管控平台，解决企业内部员工私自安装 OpenClaw 导致的数据泄露与违规使用风险。

### 1.2 核心价值

- 🔐 **统一管控** - 发现、注册、审批所有 OpenClaw 实例
- 🛡️ **安全合规** - 策略管控、行为审计、风险告警
- 🏢 **多租户隔离** - 部门/团队独立管理，权限分离
- 🤖 **智能检测** - AI 辅助异常检测、风险评分、自动响应

### 1.3 适用场景

| 场景 | 说明 |
|------|------|
| 企业安全管控 | 防止员工私自安装 OpenClaw 导致数据泄露 |
| 合规审计 | 满足等保 2.0、GDPR 等合规要求 |
| 多部门管理 | 不同部门独立管理，权限隔离 |
| 风险监控 | 实时监控 OpenClaw 使用风险 |

## 3. 安装部署

### 3.1 环境要求

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 2 核 | 4 核 |
| 内存 | 4GB | 8GB |
| 存储 | 20GB | 50GB SSD |
| 系统 | Linux/macOS/Windows | Ubuntu 22.04 LTS |
| Python | 3.11+ | 3.11+ |

### 3.2 快速安装

#### 方法一：一键启动（推荐）

**macOS/Linux:**
```bash
# 1. 解压项目
unzip openclaw-enterprise-project.zip
cd openclaw-enterprise

# 2. 启动服务
chmod +x start.sh
./start.sh

# 3. 访问系统
# 浏览器打开：http://localhost:8000/pages/dashboard
```

**Windows:**
```bash
# 1. 解压项目
# 2. 双击 start.bat
# 3. 访问 http://localhost:8000/pages/dashboard
```

#### 方法二：手动安装

```bash
# 1. 进入后端目录
cd openclaw-enterprise/backend

# 2. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate  # Linux/macOS
venv\Scripts\activate     # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 启动服务
python app/main.py

# 5. 访问系统
# 浏览器打开：http://localhost:8000/pages/dashboard
```

### 3.3 Docker 部署

```bash
# 1. 构建镜像
cd openclaw-enterprise
docker build -t openclaw/enterprise:latest .

# 2. 运行容器
docker run -d \
  -p 8000:8000 \
  -v ./data:/app/data \
  --name openclaw-enterprise \
  openclaw/enterprise:latest

# 3. 访问系统
# http://localhost:8000/pages/dashboard
```

**Docker Compose:**
```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data
    restart: unless-stopped
```

### 3.4 生产环境部署

#### 3.4.1 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

#### 3.4.2 HTTPS 配置

```bash
# 使用 Let's Encrypt
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

#### 3.4.3 系统服务（systemd）

```ini
[Unit]
Description=OpenClaw Enterprise
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/openclaw-enterprise/backend
Environment="PATH=/opt/openclaw-enterprise/backend/venv/bin"
ExecStart=/opt/openclaw-enterprise/backend/venv/bin/python app/main.py
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
# 启用服务
sudo systemctl enable openclaw
sudo systemctl start openclaw
sudo systemctl status openclaw
```

### 3.5 终端 Agent 安装

在需要监控的目标设备上安装 Agent：

**步骤 1: 下载安装脚本**

1. 访问 `http://服务器地址/pages/agent`
2. 填写服务器地址
3. 点击"下载安装脚本"
4. 在目标设备运行脚本

**步骤 2: 手动安装**

```bash
# 1. 下载并运行安装脚本
bash install-agent.sh

# 2. 启动 Agent
python3 agent_daemon.py --config ~/.openclaw-enterprise/agent-config.json

# 3. 注册开机自启（可选）
# macOS: 自动注册 launchd
# Linux: 自动注册 systemd
```

## 4. 快速入门

### 4.1 首次登录

1. 访问 `http://localhost:8000/pages/dashboard`
2. 默认无需登录（生产环境需配置认证）
3. 进入控制台总览页面

### 4.2 核心操作流程

#### 场景 1: 发现并管控私自安装的 OpenClaw

**步骤 1: 网络扫描**
1. 点击左侧导航 **🔍 网络扫描**
2. 选择扫描模式（快速/标准/深度）
3. 配置扫描端口（默认 8000,8080,3000）
4. 点击"开始扫描"
5. 查看扫描结果

**步骤 2: 导入设备**
1. 勾选发现的设备
2. 点击"导入到资产"
3. 填写设备信息（用户、部门）
4. 点击"提交"

**步骤 3: 阻断违规设备**
1. 找到状态为"违规"的设备
2. 点击"阻断"按钮
3. 确认阻断操作
4. 设备被加入黑名单

#### 场景 2: 配置安全策略

**步骤 1: 访问策略中心**
1. 点击左侧导航 **🛡️ 策略中心**

**步骤 2: 配置策略**
1. 点击策略开关启用/禁用
2. 点击策略名称进入详情
3. 修改配置参数
4. 点击"保存配置"

**默认策略:**
- 🚫 禁止访问敏感目录
- 🔑 强制密钥管理
- 🌐 限制外部模型连接
- 📦 第三方技能沙箱
- ⚠️ 高危命令需审批
- 📊 操作日志实时上报
- 🔄 自动版本升级

#### 场景 3: 查看审计日志

**步骤 1: 访问审计日志**
1. 点击左侧导航 **📋 审计日志**

**步骤 2: 筛选日志**
1. 选择操作类型（全部/文件/命令/API/消息）
2. 设置时间范围
3. 选择设备/用户
4. 点击"应用"

**步骤 3: 导出日志**
1. 点击"导出 CSV"
2. 选择导出范围
3. 文件自动下载

#### 场景 4: 处理告警

**步骤 1: 查看告警**
1. 点击左侧导航 **🚨 告警中心**
2. 查看告警列表

**步骤 2: 处理告警**
1. 点击告警行查看详情
2. 点击"标记解决"
3. 填写解决说明
4. 确认提交

## 7. 故障排查

### 7.1 服务无法启动

**问题:** 服务启动失败

**排查步骤:**
```bash
# 1. 检查端口占用
lsof -i :8000

# 2. 检查日志
tail -f openclaw.log

# 3. 检查 Python 版本
python3 --version

# 4. 重新安装依赖
pip install -r requirements.txt --force-reinstall
```

### 7.2 数据库连接失败

**问题:** 数据库无法连接

**排查步骤:**
```bash
# 1. 检查数据库文件
ls -la *.db

# 2. 检查文件权限
chmod 644 openclaw.db

# 3. 重建数据库
rm openclaw.db
python app/main.py  # 自动重建
```

### 7.3 前端无法访问

**问题:** 前端页面打不开

**排查步骤:**
```bash
# 1. 检查后端服务
curl http://localhost:8000/api/stats

# 2. 检查 Nginx 配置
nginx -t

# 3. 检查 Nginx 日志
tail -f /var/log/nginx/error.log

# 4. 重启 Nginx
sudo systemctl restart nginx
```

### 7.4 扫描失败

**问题:** 网络扫描无结果

**排查步骤:**
```bash
# 1. 检查网络连通性
ping 目标设备

# 2. 检查防火墙
sudo iptables -L

# 3. 检查端口
nmap -p 8000 目标 IP

# 4. 增加扫描超时时间
# 在扫描配置中调整 timeout 参数
```

### 7.5 阻断失败

**问题:** 阻断操作失败

**排查步骤:**
```bash
# 1. 检查 sudo 权限
sudo -v

# 2. 检查 iptables
sudo iptables -L

# 3. 手动测试阻断
sudo iptables -A OUTPUT -d 目标 IP -j DROP

# 4. 查看执行日志
tail -f openclaw.log
```

## 8. 常见问题

### 8.1 安装部署

**Q: 忘记密码怎么办？**
A: 点击登录页"忘记密码"，通过注册邮箱重置密码。

**Q: 账号被锁定怎么办？**
A: 联系系统管理员解锁账号。

**Q: 支持哪些操作系统？**
A: 支持 Linux、macOS、Windows。推荐 Ubuntu 22.04 LTS。

**Q: 需要多少资源？**
A: 最低 2 核 CPU、4GB 内存、20GB 存储。推荐 4 核 CPU、8GB 内存、50GB SSD。

### 8.2 设备管理

**Q: 设备扫描不到怎么办？**
A: 
1. 确认设备已安装 OpenClaw
2. 确认设备与服务器网络连通
3. 确认设备防火墙未阻挡扫描

**Q: 设备阻断后如何恢复？**
A: 
1. 进入设备详情
2. 点击"解除阻断"
3. 填写解除原因
4. 管理员审批后恢复

**Q: 如何批量导入设备？**
A: 
1. 网络扫描发现设备
2. 勾选需要导入的设备
3. 点击"导入到资产"
4. 填写设备信息

### 8.3 策略配置

**Q: 策略修改后多久生效？**
A: 策略保存后立即生效，设备下次心跳时同步策略。

**Q: 如何给特定租户设置独立策略？**
A: 
1. 进入租户管理
2. 选择目标租户
3. 点击"策略"标签
4. 配置租户独立策略

**Q: 策略冲突怎么办？**
A: 全局策略优先级高于租户策略。

### 8.4 审计日志

**Q: 日志最多保留多久？**
A: 默认保留 180 天，可在系统设置中调整。

**Q: 导出的日志文件在哪里？**
A: 默认下载到浏览器指定的下载目录。

**Q: 如何查询特定用户的操作？**
A: 
1. 进入审计日志
2. 选择用户筛选
3. 选择目标用户
4. 点击"应用"

### 8.5 告警处理

**Q: 告警太多处理不过来怎么办？**
A: 
1. 优先处理严重和高危告警
2. 使用批量解决功能
3. 配置告警自动解决规则

**Q: 如何避免告警风暴？**
A: 
1. 配置告警聚合规则
2. 设置告警静默时间
3. 调整告警阈值

**Q: 告警通知收不到怎么办？**
A: 
1. 检查通知配置是否正确
2. 测试通知渠道
3. 检查网络连接
4. 查看通知日志

### 8.6 系统性能

**Q: 系统响应慢怎么办？**
A: 
1. 检查服务器资源使用情况
2. 清理过期日志
3. 优化数据库索引
4. 增加服务器资源

**Q: 支持多少设备并发？**
A: 单实例支持 1000+ 设备并发，大规模部署建议使用集群模式。

**Q: 如何备份数据？**
A: 
```bash
# SQLite 备份
cp openclaw.db openclaw.db.backup.$(date +%Y%m%d)

# PostgreSQL 备份
pg_dump openclaw > backup.sql
```

4. 查看 Agent 日志
