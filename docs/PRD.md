# OpenClaw Enterprise - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
企业级 OpenClaw 集中管控平台，解决企业内部员工私自安装 OpenClaw 导致的数据泄露与违规使用风险。

### 1.2 目标客户
- 中大型企业 IT 部门
- 安全合规部门
- 研发团队管理者

### 1.3 核心价值
- **统一管控** - 发现、注册、审批所有 OpenClaw 实例
- **安全合规** - 策略管控、行为审计、风险告警
- **多租户隔离** - 部门/团队独立管理，权限分离

---

## 2. 功能需求

### 2.1 资产管理
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 设备发现 | 自动扫描内网中的 OpenClaw 实例 | P0 |
| 手动注册 | 管理员手动添加设备 | P0 |
| 设备审批 | 审批新注册的设备 | P0 |
| 设备阻断 | 远程阻断违规设备 | P0 |
| 版本管理 | 查看和强制升级设备版本 | P1 |
| 设备分组 | 按部门/项目分组管理 | P2 |

### 2.2 策略中心
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 敏感目录禁止 | 阻止访问 /etc, /root, /credentials 等 | P0 |
| 密钥管理 | 强制使用企业密钥管理系统 | P0 |
| 外部模型限制 | 限制连接未经审批的 LLM | P0 |
| 技能沙箱 | 第三方技能在隔离环境运行 | P1 |
| 高危命令审批 | rm, dd 等命令需审批 | P0 |
| 日志上报 | 实时上报到 SIEM | P1 |
| 自动升级 | 维护窗口自动升级版本 | P2 |

### 2.3 行为审计
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 文件访问记录 | 记录所有文件读写操作 | P0 |
| 命令执行记录 | 记录所有 shell 命令 | P0 |
| API 调用记录 | 记录所有 API 请求 | P0 |
| 消息发送记录 | 记录所有消息发送 | P1 |
| 日志筛选 | 按时间/设备/操作类型筛选 | P1 |
| 日志导出 | 导出 CSV/PDF 格式 | P2 |

### 2.4 告警中心
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 实时告警 | 未授权设备、敏感访问等 | P0 |
| 告警分级 | 严重/高危/中危/低危 | P0 |
| 告警通知 | 邮件/短信/飞书通知 | P1 |
| 告警解决 | 标记告警已解决 | P0 |
| 告警统计 | 按级别/时间统计 | P2 |

### 2.5 多租户管理
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 租户创建 | 创建部门/团队租户 | P0 |
| 租户隔离 | 数据/策略完全隔离 | P0 |
| 角色管理 | 管理员/审计员/普通用户 | P0 |
| 资源配额 | 限制设备数/API 调用等 | P2 |

## 3. 技术架构

### 3.1 架构设计
```
┌─────────────────────────────────────────────────────┐
│                    前端层                            │
│  (HTML/CSS/JS - 6 个功能页面)                        │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    API 网关                          │
│  (FastAPI - RESTful API)                            │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    业务层                            │
│  (设备管理、策略管理、审计、告警、租户)              │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    数据层                            │
│  (SQLite/PostgreSQL + Redis 缓存)                   │
└─────────────────────────────────────────────────────┘
```

### 3.2 技术栈
- **后端**: Python 3.11+ / FastAPI / SQLAlchemy
- **前端**: HTML5 / CSS3 / JavaScript (ES6+)
- **数据库**: SQLite (开发) / PostgreSQL (生产)
- **缓存**: Redis (可选)
- **部署**: Docker / Kubernetes

## 4. 数据模型

### 4.1 核心表结构

**Tenant (租户)**
- id, name, code, description, created_at, is_active

**User (用户)**
- id, username, real_name, email, tenant_id, role, is_active

**Device (设备)**
- id, name, hostname, tenant_id, user_id, version, status, risk_level, risk_score, last_active, created_at

**AuditLog (审计日志)**
- id, device_id, user_id, operation_type, operation_detail, risk_level, status, timestamp

**Alert (告警)**
- id, title, description, level, device_id, is_resolved, created_at, resolved_at

**SecurityPolicy (安全策略)**
- id, name, description, is_enabled, config, updated_at

## 5. API 设计

### 5.1 设备管理
- `GET /api/devices` - 获取设备列表
- `POST /api/devices` - 创建设备
- `GET /api/devices/{id}` - 获取设备详情
- `PUT /api/devices/{id}` - 更新设备
- `DELETE /api/devices/{id}` - 删除设备
- `POST /api/devices/{id}/block` - 阻断设备
- `POST /api/devices/scan` - 扫描网络

### 5.2 审计日志
- `GET /api/audit-logs` - 获取日志列表
- `GET /api/audit-logs/export` - 导出日志

### 5.3 告警管理
- `GET /api/alerts` - 获取告警列表
- `PUT /api/alerts/{id}/resolve` - 解决告警

### 5.4 策略管理
- `GET /api/policies` - 获取策略列表
- `PUT /api/policies/{id}` - 更新策略

### 5.5 租户管理
- `GET /api/tenants` - 获取租户列表
- `POST /api/tenants` - 创建租户
- `PUT /api/tenants/{id}` - 更新租户

### 5.6 统计接口
- `GET /api/stats` - 获取统计数据
- `GET /api/stats/trends` - 获取趋势数据

## 7. 安全设计

### 7.1 认证授权
- JWT Token 认证
- RBAC 角色权限控制
- API 访问频率限制

### 7.2 数据安全
- 敏感数据加密存储
- HTTPS 传输加密
- 审计日志防篡改

### 7.3 合规要求
- 等保 2.0 三级
- GDPR 数据保护
- 审计日志保留 180 天

## 8. 部署方案

### 8.1 开发环境
```bash
./start.sh  # 一键启动
```

### 8.2 生产环境
```yaml
# Docker Compose
services:
  app:
    image: openclaw/enterprise:latest
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://...
      - REDIS_URL=redis://...
  
  db:
    image: postgres:15
  
  redis:
    image: redis:7
```
