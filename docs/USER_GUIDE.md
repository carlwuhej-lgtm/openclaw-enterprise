## 功能详解

### 1. Agent 管理

#### 创建 Agent

1. 进入 **资产管理 → Agent 管理**
2. 点击 **➕ 创建 Agent**
3. 填写信息：
   - Agent ID：唯一标识（如 `daily`）
   - 名称：显示名称（如 `日常助手`）
   - 模型：选择的 AI 模型
   - 工作目录：Agent 配置目录

4. 点击 **创建**

#### 启动/停止 Agent

- **启动**: 点击 Agent 卡片上的 ▶️ 按钮
- **停止**: 点击 ⏹️ 按钮
- **重启**: 先停止再启动

#### 查看 Agent 状态

状态说明：
- 🟢 **运行中** - Agent 正常工作
- 🟡 **已停止** - Agent 已停止
- 🔴 **错误** - Agent 异常，查看日志

---

### 2. 安全策略

#### 创建安全策略

1. 进入 **安全管控 → 安全策略**
2. 点击 **➕ 创建策略**
3. 配置策略：
   - 名称：策略名称
   - 类型：文件访问/命令执行/API 调用
   - 规则：允许/拒绝条件
   - 动作：允许/拒绝/告警

4. 点击 **保存**

#### 策略示例

**禁止访问敏感目录**:
```
类型：文件访问
规则：路径包含 ~/.ssh、~/.gnupg
动作：拒绝 + 告警
```

**限制外部 LLM 调用**:
```
类型：API 调用
规则：域名不在白名单
动作：拒绝 + 告警
```

### 3. 日志中心

#### 查看日志

1. 进入 **监控审计 → 日志中心**
2. 选择时间范围
3. 使用过滤条件：
   - 主机名
   - 严重性
   - 设施类型
   - 关键词搜索

#### 导出日志

1. 选择要导出的日志
2. 点击 **📥 导出**
3. 选择格式：CSV / PDF / HTML
4. 下载文件

#### 配置日志转发

1. 进入 **日志中心 → 转发配置**
2. 点击 **➕ 添加转发目标**
3. 填写：
   - 名称：如 `公司 ELK`
   - 地址：ELK 服务器 IP
   - 端口：514（或自定义）
   - 协议：UDP/TCP

4. 点击 **保存**

---

### 4. 急救室

#### 自动诊断

1. 进入 **系统工具 → 急救室**
2. 点击 **🔍 诊断问题**
3. 查看诊断结果

#### 一键修复

1. 诊断完成后
2. 点击 **🛠️ 一键修复**
3. 系统自动：

#### 查看修复历史

- 修复记录保存在：`~/.openclaw/backups/`
- 每次修复前自动备份

## 常见问题

### Q1: 忘记密码怎么办？

**解决方法**:
```bash
# 连接到数据库
psql -U openclaw -d openclaw_enterprise

# 重置 admin 密码
UPDATE users SET hashed_password='$2b$12$...' WHERE username='admin';
```

或联系技术支持。

---

### Q2: Agent 无法启动？

**排查步骤**:
1. 检查 Gateway 状态：`openclaw gateway status`
2. 查看 Agent 日志：`tail -f /tmp/openclaw/agent-xxx.log`
3. 检查配置：`cat ~/.openclaw/agents/xxx/openclaw.json`
4. 使用急救室诊断

---

### Q3: 日志中心没有数据？

**可能原因**:
1. Syslog 服务未启动
2. 端口被占用
3. 防火墙阻止

**解决方法**:
1. 进入日志中心
2. 点击 **▶️ 启动服务**
3. 检查端口：`netstat -anp | grep 1514`
4. 配置 rsyslog 转发

---

### Q4: 数据库连接失败？

**排查步骤**:
```bash
# 1. 检查 PostgreSQL 状态
sudo systemctl status postgresql

# 2. 测试连接
psql postgresql://openclaw:密码@127.0.0.1:5432/openclaw_enterprise -c "SELECT 1;"

# 3. 查看日志
sudo tail -f /var/log/postgresql/postgresql-*.log
```

---

### Q5: 系统运行缓慢？

**优化建议**:
1. 清理旧日志（保留 90 天）
2. 数据库 VACUUM
3. 增加服务器资源
4. 检查是否有异常进程

## 最佳实践

### 1. 账号安全

- ✅ 首次登录立即修改密码
- ✅ 使用强密码（12 位+，包含特殊字符）
- ✅ 定期更换密码（90 天）
- ✅ 启用双因素认证（如支持）
- ❌ 不要共享账号
- ❌ 不要使用默认密码

### 2. 备份策略

**每日备份**:
```bash
# 添加到 crontab
0 2 * * * pg_dump postgresql://openclaw:密码@127.0.0.1:5432/openclaw_enterprise | gzip > /backup/openclaw_$(date +\%Y\%m\%d).sql.gz
```

**备份保留**:
- 每日备份：保留 30 天
- 每周备份：保留 12 周
- 每月备份：保留 12 个月

### 3. 监控告警

**建议监控项**:
- CPU 使用率 > 80%
- 内存使用率 > 85%
- 磁盘使用率 > 90%
- 数据库连接数 > 80%
- Agent 离线超过 10 分钟

### 4. 日志管理

**日志保留策略**:
- 审计日志：90 天
- Syslog：30 天
- 操作日志：180 天
- 错误日志：永久

**定期清理**:
```sql
-- 每月执行一次
DELETE FROM audit_logs WHERE timestamp < NOW() - INTERVAL '90 days';
VACUUM ANALYZE;
```

### 5. 性能优化

**数据库优化**:
- 定期 VACUUM（每周）
- 添加索引（审计日志时间字段）
- 连接池配置（max_connections=100）

**应用优化**:
- 启用缓存（Redis）
- 静态资源 CDN
- 分页查询（每页≤100 条）
