## 产品简介

### 什么是 OpenClaw Enterprise？

OpenClaw Enterprise 是企业级 AI Agent 安全治理平台，用于集中管理和监控企业内的 OpenClaw 智能体。

### 核心功能

- **实例管理** - 注册、发现和监控所有 OpenClaw 实例
- **Agent 管理** - 管理 AI 智能体的生命周期
- **安全管控** - 策略引擎、操作拦截、风险告警
- **审计日志** - 完整的操作记录和追溯
- **合规报告** - PDF/CSV 格式的安全报告

### 适用角色

| 角色 | 职责 | 权限 |
|------|------|------|
| **系统管理员** | 系统配置、用户管理 | 全部权限 |
| **安全管理员** | 策略配置、告警处理 | 安全管理权限 |
| **审计员** | 日志查看、报告导出 | 只读权限 |
| **普通用户** | 查看自己管理的设备 | 有限权限 |

## 快速入门

### 1. 登录系统

访问 `http://your-server:8003`，使用管理员账号登录。

**默认账号**（首次部署）:
- 用户名：`admin`
- 密码：创建时设置

### 2. 注册第一个实例

1. 进入 **实例管理** 页面
2. 点击 **➕ 注册实例**
3. 填写实例信息：
   - 实例名称：如 "开发部 - 张三的电脑"
   - 主机名：自动填充
   - IP 地址：自动填充
   - 位置：选择 LAN 或 WAN
4. 点击 **创建**

### 3. 安装 Agent

在实例上执行以下命令：

```bash
# 下载并安装
curl -sSL https://your-server:8003/install.sh | sudo bash

# 验证安装
systemctl status openclaw-agent
```

### 4. 配置安全策略

1. 进入 **安全管控** 页面
2. 点击 **➕ 创建策略**
3. 选择策略类型：
   - 敏感目录禁止
   - 高危命令审批
   - 外部模型限制
4. 配置策略规则
5. 启用策略

### 5. 查看审计日志

1. 进入 **审计日志** 页面
2. 使用筛选条件：
   - 时间范围
   - 操作类型
   - 风险级别
3. 点击 **📥 导出** 生成报告

## 功能使用

### 实例管理

#### 注册实例

**手动注册**:
1. 进入 **实例管理**
2. 点击 **➕ 注册实例**
3. 填写信息并保存

**自动发现**:
1. 点击 **🔍 扫描局域网**
2. 选择扫描模式（快速/标准/深度）
3. 选择发现的实例并导入

#### 查看实例详情

点击实例列表中的 **详情** 按钮，查看：
- 基本信息（名称、主机名、IP）
- 系统信息（OS、CPU、内存、磁盘）
- OpenClaw 状态
- Agent 列表
- 风险评分趋势

#### 批量操作

1. 勾选需要操作的实例
2. 点击顶部的批量操作按钮：
   - **批量删除** - 删除选中的实例
   - **批量阻断** - 阻断选中的实例
   - **批量解除** - 解除阻断

### Agent 管理

#### 创建 Agent

1. 进入 **Agent 管理**
2. 点击 **➕ 创建 Agent**
3. 填写信息：
   - Agent ID：如 `daily`, `security`
   - 名称：显示名称
   - 模型：如 `bailian/glm-5`
   - 工作目录：Agent 工作路径
4. 点击 **创建**

#### 启动/停止 Agent

- **启动**: 点击 Agent 列表中的 **▶️ 启动**
- **停止**: 点击 **⏹️ 停止**
- **重启**: 先停止再启动

#### 查看 Agent 状态

状态说明：
- 🟢 **running** - 正常运行
- 🔴 **stopped** - 已停止
- 🟡 **error** - 异常状态
- ⚪ **unknown** - 未知状态

### 安全管控

#### 创建策略

1. 进入 **安全管控**
2. 点击 **➕ 创建策略**
3. 选择策略类型

**内置策略模板**:
- 敏感目录保护
- 高危命令拦截
- API 密钥防泄露
- 渗透工具检测

4. 配置策略参数
5. 选择生效范围（全部实例/指定实例）
6. 启用策略

#### 策略优先级

策略按以下顺序执行：
1. 阻断类策略（最高优先级）
2. 审批类策略
3. 告警类策略
4. 记录类策略（最低优先级）

### 告警中心

#### 查看告警

告警列表显示：
- 告警标题
- 风险级别（严重/高/中/低）
- 关联实例
- 状态（未解决/已解决）
- 创建时间

#### 处理告警

**单个解决**:
1. 找到告警
2. 点击 **✅ 解决**
3. 填写解决说明（可选）

**批量解决**:
1. 勾选多个告警
2. 点击 **✅ 批量解决**

#### 告警级别说明

| 级别 | 颜色 | 说明 | 示例 |
|------|------|------|------|
| **严重** | 🔴 | 可能导致系统崩溃或数据丢失 | `rm -rf /` |
| **高** | 🟠 | 可能导致配置破坏或信息泄露 | 修改 SSH 配置 |
| **中** | 🟡 | 可疑行为，需要关注 | 访问敏感文件 |
| **低** | 🔵 | 轻微风险，信息性告警 | 读取 /etc/passwd |

### 审计日志

#### 筛选日志

支持以下筛选条件：
- **时间范围**: 开始时间 - 结束时间
- **操作类型**: 文件操作/命令执行/API 调用
- **风险级别**: 严重/高/中/低/安全
- **状态**: 放行/拦截/告警
- **设备**: 选择特定实例

#### 导出日志

1. 设置筛选条件
2. 点击 **📥 导出**
3. 选择格式：
   - **CSV** - Excel 可打开
   - **PDF** - 直接打印
   - **HTML** - 网页格式
4. 下载文件

### 分析报告

#### 流量分析

查看：
- 审计日志总数
- LLM 调用次数
- 已拦截操作数
- 24 小时趋势图

#### AI 辅助

功能：
- 风险评分排行
- 异常行为检测
- 行为画像分析
- 自动响应配置

#### 报表导出

**定期报告**:
- 日报（每日生成）
- 周报（每周一生成）
- 月报（每月 1 号生成）

**自定义报告**:
1. 选择数据类型
2. 设置日期范围
3. 选择导出格式
4. 下载报告

## 常见问题

### Q1: 实例显示离线怎么办？

**可能原因**:
1. 实例网络中断
2. Agent 服务停止
3. 防火墙阻挡

**解决方法**:
```bash
# 1. 检查网络
ping your-server:8003

# 2. 检查服务状态
systemctl status openclaw-agent

# 3. 重启服务
sudo systemctl restart openclaw-agent
```

### Q2: 如何添加新用户？

1. 进入 **租户管理**
2. 选择对应租户
3. 点击 **➕ 创建用户**
4. 填写用户信息
5. 分配角色

### Q3: License 过期了怎么办？

**查看 License 状态**:
1. 点击右上角用户头像
2. 选择 **License 信息**
3. 查看过期时间

**续费/升级**:
- 联系销售团队
- 在线购买（如支持）
- 申请试用延期

### Q4: 误拦截了正常操作怎么办？

**方法 1: 添加白名单**
1. 进入 **安全管控**
2. 找到对应策略
3. 添加白名单规则

**方法 2: 调整策略**
1. 编辑策略
2. 放宽规则
3. 保存并测试

### Q5: 如何备份数据？

**数据库备份**:
```bash
# PostgreSQL
pg_dump -h localhost -U openclaw openclaw_enterprise > backup.sql

# SQLite
cp openclaw.db openclaw.backup.db
```

**配置文件备份**:
```bash
tar -czf backup-config.tar.gz backend/.env backend/license.json
```

### A. API 快速参考

**认证**:
```bash
curl -X POST http://localhost:8003/api/token \
  -d "username=admin&password=your_password"
```

**获取设备列表**:
```bash
curl http://localhost:8003/api/devices \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**创建告警**:
```bash
curl -X POST http://localhost:8003/api/alerts \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"测试告警","level":"high"}'
```
